#define IRFENCE   80

A memory fence. This is an instruction for ordering other memory operations.

The first parameter decides the type of fence.

Options are: 0 = NotAtomic 1 = Unordered 2 = Monotonic 3 = Acquire 4 = Release 5 = AcquireRelease 6 = SequentiallyConsistent
