Macro Definition Documentation

#define IRCLASSID   18

Represent the identificator of an application class (e.g. CIL class).

#define IRFPOINTER   23

Function pointer

#define IRMETHODENTRYPOINT   20

A IR variable (or constant) of this kind reports the first address where the assembly of a IR method is stored.
If that IR method is not yet compiled, it reports the address where the assembly of the trampoline to that method is stored.

#define IRMETHODID   19

Represent the identificator of an application method (e.g. CIL method, IR method).
A IR variable (or constant) of this type (stored inside the internal_type field of ir_item_t) will store the identificator of a IR method.

#define IRMPOINTER   21

Pointer to objects or fields of objects (Managed pointer).
The difference with IROBJECT is that IRMPOINTER can additionally point (i.e., store the address) to a field of an object rather than pointing only to the first byte of the object as IROBJECT does.

#define IROBJECT   24

IROBJECT represents Application object (instances of application classes)

#define IRSIGNATURE   27

A IR variable (or constant) of this type stores the memory address of the t_ir_signature .

#define IRSTRING   26

Represent strings of the compiler. Basically they are pointer to char.
Notice that they do not represent Application strings (e.g. CIL strings) which are of type IROBJECT.
