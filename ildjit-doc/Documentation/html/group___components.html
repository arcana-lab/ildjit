Overview
Collaboration diagram for Components:

ILDJIT is composed by a framework and by external plugins that you can customize or extend for your specific purposes.

The framework provides a set of APIs the plugins can rely on, which are the following:

  • System API : ILDJIT system interface
  • Input language API : abstraction of supported input languages
  • IR API : how to understand and manipulate the IR language
  • Codetool API : how to interact with codetools
  • Platform API : abstraction of the underlying platform
  • Memory API : how to manage the memory
  • Data structure API : data structures

Different type of plugins can be built within the ILDJIT framework, which are the following:

  • Codetools : plugins that analyze and/or transform IR code.
  • Optimization level tools : these plugins define the optimization levels. They decide which optimizations to run for each optimization level.
  • Garbage collection tools : these plugins implement the garbage collection for memory used by the IR code.
  • CLR tools : these plugins provide methods in their native form (e.g. native methods of the base class library – BCL).
  • Backend tools : these plugins provide the translation from IR to the machine code.

Each type of plugin can access different set of APIs
