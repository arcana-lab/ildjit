Detailed Description

Data can be stored using the IR language in:

  • method variables
  • constants
  • symbols
  • globals

All data is represented by instances of ir_item_t . How to manage these data types is described in Data type .

In more details, IR variables, constants, symbols, and globals (i.e., ir_item_t ) have all five fields: value, fvalue, type, internal_type, type_infos.

  • type : This field defines if this element is a variable, a constant, a symbol, or a global. If the IR item is a variable, type is equal to IROFFSET. If it is a constant, then this field is equal to internal_type . Moreover, if it is a symbol, then it is equal to IRSYMBOL. Finally, if it is a global element (either variable or constant), then it is equal to IRGLOBAL.
  • internal_type : this is the actual builtin type of the IR item (e.g., IRINT32 )
  • value : if the item is a constant, then this field stores the value of that constant. If the item is a variable (i.e., type is equal to IROFFSET), then this field stores the identificator of the variable (i.e., its ID). If the item is a symbol, then this field points to the obfuscated data structure that internal describe the symbol (to use the symbol, use the API describes in Symbols).
  • fvalue : This field is useful only for floating point constants. In this case, this field stores the floating point value of that constant.
  • type_infos : this field reports the metadata description of the high level data type of the IR variable or constant (i.e., class description)

For example, the variable 5 which stores an integer 32 bits value will have

  • type = IROFFSET
  • internal_type = IRINT32
  • value = 5
  • type_infos = high level metadata description of the integer 32 bits type

Moreover, the integer 32 bits constant equal to 425 will have

  • type = IRINT32
  • internal_type = IRINT32
  • value = 425
  • type_infos = high level metadata description of the integer 32 bits type

Finally, for the floating point 32 bits constant equal to 5.123, we will have

  • type = IRFLOAT32
  • internal_type = IRFLOAT32
  • fvalue = 5.123
  • type_infos = high level metadata description of the floating point 32 bits type

For example, if the variable has the value equal to IRINT8 (i.e. ir_item_t.value == IRINT8) it means that identify the IRINT8 type.
