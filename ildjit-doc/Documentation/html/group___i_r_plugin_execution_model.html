This section provides information on how every plugin are called every time you run ILDJIT. The interface between the IR code optimization plugin and ILDJIT is defined within the file libiljitiroptimizer/src/ir_optimization_interface.h .

The IR code optimization plugins have to implement the following functions:

JITUINT64 (*get_job_kind) (void);
JITUINT64 (*get_dependences) (void);
void (*init) (ir_lib_t *irLib, ir_optimizer_t *optimizer, char *outputPrefix);
void (*shutdown) (void);
void (*do_job) (ir_method_t *method);
char * (*get_version) (void);
char * (*get_information) (void);
char * (*get_author) (void);
JITUINT64 (*get_invalidations) (void);

The function init is called when ILDJIT boosts and hence only once. On the other hand, the function shutdown is called just before ILDJIT shuts down the system (hence only once).

The get_job_kind , get_invalidations and get_dependences functions are called every time ILDJIT needs for its purposes.

Finally, the function do_job is called every time a IR method needs to be optimized. Hence, this last function may be called multiple times.

Every plugin can call every function provided by ILDJIT and defined within the IR API module described later in order to manage the IR code. On the other hand, in order to call other plugins, ILDJIT provides the Codetool API.

Enabling and Disabling plugins

ILDJIT has a list of optimizations to use for every optimization level. If you want to change this list, you can enable (i.e., –enable-optimizations=...) or disable (i.e., –disable-optimizations=...) optimizations by command line.

If you use the "--enable-optimizations=" option, then ILDJIT will run only the optimizations you specify in the command line.

On the other hand, if you use the "--disable-optimizations=" option, then ILDJIT consider every plugin activated by default minus the ones you specify in the command line.

When ILDJIT boosts, it will load every plugin is able to find on the default installation directory (i.e., INSTALLATION_PREFIX/lib/iljit/optimizers ). If you would like to customize this behavior, you can set the environment variable ILDJIT_PLUGINS to point to a list of directories where you have placed your own plugins.

For example, consider the following case:

export ILDJIT_PLUGIN=/home/xan/quite_stable_plugins:/home/xan/experimental_plugins:/home/xan/completely_crazy_plugins

In this example, ILDJIT will loads every plugins installed in the following directories:

  • INSTALLATION_PREFIX/lib/iljit/optimizers
  • /home/xan/quite_stable_plugins
  • /home/xan/experimental_plugins
  • /home/xan/completely_crazy_plugins
