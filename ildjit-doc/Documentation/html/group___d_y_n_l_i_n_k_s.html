Close shared object.

This function is used to inform the system that the object referenced by a shared object handle is no longer needed by the application.

void* PLATFORM_dlopen ( const char *  arg1,
JITNINT  arg2 
)

Gain access to an executable object file.

This function makes an executable object file specified by file available to the calling program. (see dlopen())

Parameters
fileSpecifies the object file to load. If file is not an absolute path, an implementation-defined algorithm is used to locate the file. If file is NULL, dlopen() provides a handle to the calling program itself.
modeSpecifies the mode in which symbols should be bound. This may be either RTLD_LAZY or RTLD_NOW. These values are interpreted identically; symbols are always bound at load time. Both are recognized for maximum source compatibility, since there is no operational difference.
Returns
If success, returns a pointer to dynamic link oject. Returns NULL in case of error (call PLATFORM_dlerror() for more detailed diagnostic informations)

Obtain the object of the symbolic link. This function allows a process to obtain the address of a symbol defined within an object.
(see dlsym())

Parameters
fileSpecifies the object file to load. If file is not an absolute path, an implementation-defined algorithm is used to locate the file. If file is NULL, dlopen() provides a handle to the calling program itself.
modeSpecifies the mode in which symbols should be bound. This may be either RTLD_LAZY or RTLD_NOW. These values are interpreted identically; symbols are always bound at load time. Both are recognized for maximum source compatibility, since there is no operational difference.
Returns
Returns the address binding of the symbol as it occurs in the shared object identified by handle. It returns a NULL pointer if the symbol cannot be found. More detailed diagnostic information is available through PLATFORM_dlerror()
